# منصة الامتحانات الإلكترونية — تسجيل جلسة الـ Brainstorm

---

## 1. نظرة عامة على المنصة

منصة SaaS مخصصة لإجراء الامتحانات الإلكترونية في البيئة الجامعية الجزائرية.

---

## 2. المستخدمون

- **الأستاذ** — حساب يُنشأ من طرف الإدارة (admin)
- **الطالب** — حساب يُنشأ تلقائياً عند إضافته من طرف الأستاذ

---

## 3. الهيكل التنظيمي

### المستويات (محددة في النظام مسبقاً)
- أولى / ثانية / ثالثة ليسانس
- أولى / ثانية ماستر

### الأفواج (Groups)
- الأستاذ يضيف الفوج مرتبطاً بمستوى ومادة
- مثال: مادة التاريخ / مستوى 2 ماستر / فوج D4

### المقاييس (Modules)
- بنك مقاييس مشترك في النظام
- في البداية: الأساتذة يضيفون المقاييس بحرية
- مع الوقت: الإدارة تُنقّي البنك وتجعله قائمة اختيار
- إذا لم يجد الأستاذ مقياسه في القائمة → يضيفه يدوياً

### علاقة الأستاذ بالمقاييس
```
أستاذ → يدرّس → مقاييس متعددة × مستويات × أفواج
```

---

## 4. التنقيط

### العلامات
| المكوّن | الوصف |
|---------|--------|
| علامة الامتحان | محسوبة تلقائياً من نتيجة الامتحان الإلكتروني |
| العمل الشخصي | يُدخلها الأستاذ يدوياً |
| المواضبة والسلوك | يُدخلها الأستاذ يدوياً |
| المشاركة | يُدخلها الأستاذ يدوياً |

### نظام الإعدادات
- **Template** واحد محدد في النظام مسبقاً (إلزامات إدارية)
- كل أستاذ عند إنشاء حسابه يأخذ **نسخة** من هذا الـ template
- يمكنه تعديلها إن احتاج (المجموع الكلي، وزن كل مكوّن)
- المنصة تحسب العلامة الإجمالية تلقائياً

---

## 5. الغيابات

- العتبة القصوى: **5 غيابات** → الطالب مقصى ولا يستطيع الدخول للمنصة
- هذا قرار إداري من الجامعة وليس من الأستاذ
- العتبة قابلة للضبط من إعدادات النظام (admin)

---

## 6. بنك الأسئلة

- الأسئلة مرتبطة بـ: **أستاذ + مقياس + مستوى**
- كل سؤال له تصنيف صعوبة: **سهل / متوسط / صعب**
- نوع الأسئلة: **اختيار من متعدد فقط** (MCQ)
- ترتيب الاختيارات داخل كل سؤال **عشوائي دائماً**

---

## 7. إعدادات الامتحان

الأستاذ عند إنشاء امتحان يحدد:
- المقياس والمستوى والفوج
- عدد أسئلة الامتحان
- توزيع الصعوبة: كم سؤال سهل + متوسط + صعب
- مدة الامتحان (بالدقائق)

---

## 8. خوارزمية توزيع الأسئلة

### الهدف
أقصى عشوائية ممكنة لمنع الغش مع ضمان العدالة في التوزيع.

### المنطق
```
البنك يكفي جميع الطلبة؟
  ✓ نعم → كل طالب يأخذ دفعة فريدة من الأسئلة (لا تكرار)
  ✗ لا  → نبدأ دورة جديدة مع تغيير الترتيب
```

### العشوائية على ثلاثة مستويات
1. **الأسئلة نفسها** — من البنك حسب التصنيف
2. **ترتيب الأسئلة** — عشوائي لكل طالب
3. **ترتيب الاختيارات** — عشوائي داخل كل سؤال

### في حالة نفاد البنك
- يُسمح بتكرار نفس الأسئلة بين الطلبة
- لكن الترتيب يبقى مختلفاً دائماً

---

## 9. جلسة الامتحان (تجربة الطالب)

### العرض
- جميع الأسئلة مرئية مرة واحدة (مثل Google Forms)
- الطالب يتنقل بحرية بين الأسئلة

### حفظ الإجابات
- كل إجابة تُحفظ فوراً في DB كـ **draft**
- عند التعديل تُحدَّث مباشرة
- تتحول لـ **نهائية** عند التأكيد أو انتهاء الوقت

### قفل الصفحة
- الطالب لا يستطيع فتح tab آخر أو تطبيق آخر طوال الامتحان

### التأكيد
- **تأكيد نهائي** → يُحوَّل فوراً لصفحة النتائج ولا يمكن العودة

### النتائج
- تظهر مباشرة بعد التأكيد النهائي
- الطالب يرى إجاباته وأين أخطأ

---

## 10. إدارة الوقت

### Timer
- محسوب **server-side** دائماً
- الصيغة: `deadline = started_at + duration + global_extra + student_extra`
- لا يمكن التلاعب به من طرف الطالب

### انتهاء الوقت
- الإجابات الـ draft تتحول تلقائياً لنهائية
- الطالب يُحوَّل لصفحة النتائج

### صلاحيات التمديد (للأستاذ)
| النوع | الوصف |
|-------|--------|
| تمديد جماعي | يضيف دقائق لجميع طلبة الفوج دفعة واحدة |
| تمديد فردي | يضيف دقائق لطالب معين (حالة خاصة) |

---

## 11. الحماية من انقطاع الاتصال

### المبدأ الأساسي
الشاشة مقفلة → لا يخرج أحد حتى يؤكد إجاباته نهائياً، حتى لو انقطع الاتصال.

### طبقتان للحماية
1. **Draft في DB** — عند كل اختيار مباشرة (الأساس)
2. **localStorage** — كـ buffer مؤقت فقط، ريثما يعود الاتصال والطالب لا يزال في الصفحة

### سيناريو انقطاع الاتصال
```
انقطع النت
    ↓
الشاشة تبقى مقفلة (الطالب لا يستطيع الخروج)
الإجابات محفوظة في localStorage
    ↓
الأستاذ يطلب من الطلبة الانتقال لمكان فيه تغطية
الأستاذ مسؤول عن منع الغش أثناء التنقل
    ↓
الأستاذ يضيف دقيقة إضافية
    ↓
الطلبة يصلون للشبكة → sync صامت → تأكيد نهائي → نتائج
```

### ما لا نحتاجه
- ❌ offline sync معقد
- ❌ نافذة قبول إجابات بعد انتهاء الوقت
- localStorage ليس حلاً مستقلاً بل buffer مؤقت فقط

## 15. واجهة الأستاذ أثناء الامتحان (Live Monitor)

### قائمة الطلبة تعرض لكل طالب
- الاسم
- آخر سؤال تمت الإجابة عليه
- الوقت المتبقي
- حالة الاتصال (متصل / منقطع)

### كشف انقطاع الاتصال
- عبر **heartbeat** — نبضة كل X ثوانٍ من صفحة الطالب للـ server
- إذا توقفت النبضة → الطالب يظهر كـ "منقطع"
- تنبيه **بصري + صوتي** فوري عند الأستاذ

### صلاحيات الأستاذ
- بدء الامتحان يدوياً
- إنهاء الامتحان يدوياً
- إضافة وقت جماعي لكل الفوج
- إضافة وقت فردي لطالب معين

### 👨‍🏫 الأستاذ

#### الرئيسية والإعدادات

| الصفحة | المحتوى |
|--------|---------|
| `dashboard` | نظرة عامة — عدد الطلبة، الامتحانات القادمة، آخر النتائج، إشعارات غير مقروءة |
| `profile` | تعديل البيانات الشخصية، تغيير كلمة المرور |
| `settings` | إعدادات التنقيط (نسخته الخاصة من الـ template) — المجموع الكلي، وزن كل مكوّن |
| `notifications` | قائمة كل الإشعارات |

#### المقاييس والأفواج

| الصفحة | المحتوى |
|--------|---------|
| `modules` | قائمة مقاييسه — إضافة مقياس (اختيار من البنك أو إضافة جديد)، ربطه بمستوى وفوج |
| `groups` | قائمة أفواجه — إنشاء فوج، تحديد اسمه والمستوى والمقياس المرتبط به |
| `groups/{id}` | تفاصيل فوج — قائمة طلبته، علاماتهم، غياباتهم، إضافة طالب جديد |
| `students/{id}` | ملف الطالب — بياناته، كل علاماته، سجل غياباته، نتائج امتحاناته |

#### بنك الأسئلة

| الصفحة | المحتوى |
|--------|---------|
| `questions` | قائمة كل أسئلته مع فلترة بالمقياس والمستوى والصعوبة |
| `questions/create` | إضافة سؤال — النص، الاختيارات الأربعة، الإجابة الصحيحة، تصنيف الصعوبة، المقياس والمستوى |
| `questions/{id}/edit` | تعديل سؤال موجود |

#### الامتحانات

| الصفحة | المحتوى |
|--------|---------|
| `exams` | قائمة كل امتحاناته — القادمة، الجارية، المنتهية |
| `exams/create` | إنشاء امتحان — اختيار المقياس والفوج، عدد الأسئلة، توزيع الصعوبة، المدة الزمنية |
| `exams/{id}` | تفاصيل امتحان — إعداداته، قائمة الطلبة المشاركين، حالة كل طالب |
| `exams/{id}/monitor` | **شاشة المراقبة المباشرة** — قائمة الطلبة مع آخر سؤال أجابوا عليه، حالة الاتصال، الوقت المتبقي، أزرار إضافة الوقت، بدء وإنهاء الامتحان |
| `exams/{id}/results` | نتائج الامتحان — جدول بعلامات كل الطلبة، إحصائيات، الأسئلة الأكثر إخفاقاً |

#### العلامات

| الصفحة | المحتوى |
|--------|---------|
| `grades/{group_id}` | جدول علامات فوج — إدخال علامات العمل الشخصي، المواضبة، المشاركة لكل طالب، العلامة الإجمالية المحسوبة تلقائياً |

| الصفحة | المحتوى |
